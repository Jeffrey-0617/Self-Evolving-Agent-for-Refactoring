# Static Analysis Tools

This folder stores Python tools generated by the agent during refactoring tasks. A tool is created when the agent encounters a recurring Wright# ADL failure pattern that can be checked mechanically before submitting the design to the PAT verifier.

Generated tools are not just written once and assumed to work. Each tool is tested against both positive cases and failure cases:

- **Positive cases** confirm the tool accepts valid Wright# ADL and does not report false violations.
- **Failure cases** confirm the tool catches the specific mistake pattern it was generated to detect.

The generated tool is then linked to a skill that explains when to use it, how to interpret its output, and what repair action to take when it fails.

For example, the generated port-name checker can be found at [`Evolved_skills_memory/tools/check_port_name_uniqueness/check_port_name_uniqueness.py`](check_port_name_uniqueness/check_port_name_uniqueness.py). Its linked skill can be found at [`Evolved_skills_memory/skills/check-port-name-uniqueness/SKILL.md`](../skills/check-port-name-uniqueness/SKILL.md).

That tool includes tests for valid ADL files where all port names are unique, as well as failure cases where the same port name is reused across components or repeated within a component. This makes the tool useful as a pre-verification guard: it catches a known structural error before PAT reports a less direct verification failure.
